# Руководство по внесению вклада

Спасибо за интерес к участию в разработке Crab OSINT Tool! Вот как вы можете помочь:

## Как внести свой вклад

### Сообщение об ошибках
1. Проверьте, не была ли ошибка уже зарегистрирована
2. Используйте шаблон для сообщений об ошибках
3. Включите подробную информацию для воспроизведения ошибки

### Предложение новых функций
1. Проверьте, не было ли предложение уже внесено
2. Объясните, почему эта функция полезна
3. Опишите, как она должна работать

### Внесение изменений в код
1. Форкните репозиторий
2. Создайте новую ветку для ваших изменений
3. Внесите изменения
4. Напишите тесты, если это необходимо
5. Убедитесь, что все тесты проходят
6. Создайте Pull Request

## Стандарты кода

### Стиль кодирования
- Следуйте PEP 8 для Python кода
- Используйте осмысленные имена переменных и функций
- Комментируйте сложные участки кода
- Пишите документацию для функций и классов

### Структура коммитов
- Используйте осмысленные сообщения коммитов
- Формат: `[тип]: краткое описание`
- Типы: feat, fix, docs, style, refactor, test, chore
